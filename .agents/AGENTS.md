# Coding Style Rules for this Project

- **Readability Over Cleverness:** All code written in this workspace must prioritize readability, understandability, and clear explanations. Do not use clever shortcuts, terse one-liners, or undocumented magic numbers/hacks.
- **Thorough Documentation:** Every class, function, and logical block must have docstrings and descriptive comments explaining the intent and behavior.
- **Explicit Variable Names:** Do not use cryptic abbreviations. Choose verbose, descriptive variable names.
- **Vibe Coding Safety:** Even during rapid iteration or experimental code additions (vibe coding), do not bypass these rules. Every file modification must follow clear styling and documentation standards.
- **Modern & Funky UI Design:** Any UI designed or modified in this project must follow the `funky-modern-ui` skill rules. Avoid boring standard enterprise colors and grays, avoid pure blacks (`#000000`), and prefer tasteful, vibrant, and fun neon/pastel color palettes. All buttons must be generously rounded (pill-shaped or rounded corners), and typography should utilize fun, modern Google Fonts (e.g., Outfit, Comfortaa, Fredoka) instead of standard system fonts.
